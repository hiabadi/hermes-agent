## 2024-05-19 - Replace O(N) regex evaluation with Array Index modulus
**Learning:** In React components that highlight search text, using `new RegExp(..., 'gi')` combined with `String.prototype.split(regex)` creates an array where matches and non-matches alternate perfectly (matches are always at odd indices due to the capture group in the regex). The codebase had been running `.test(part)` on every single piece of the split string array. `.test()` on a `gi` regex is stateful (maintains `lastIndex`), which can lead to bugs with repeated identical matches, and is O(N) time complexity for each block.
**Action:** Use `i % 2 === 1` instead of `regex.test(part)` after `split(regex_with_capture_group)`. It eliminates thousands of redundant RegExp operations and prevents stateful regex bugs, improving performance significantly for long log lines and chat messages with many hits.
