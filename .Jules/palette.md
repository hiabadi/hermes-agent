## 2024-11-20 - Ensure Keyboard Focus States and ARIA Labels on Icon-only Clear/Close Buttons
**Learning:** Found an accessibility issue pattern specific to this app's components: The secondary interactive elements nested inside inputs (like the "clear search" icon-only buttons) or modals (close icons) lack both ARIA descriptions and visible focus indicators, making them functionally invisible to keyboard and screen reader users. The app generally handles primary buttons well, but these utility buttons are overlooked.
**Action:** Consistently apply Tailwind utility classes like `focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring` to interactive elements and always use `t.common.clear` or similar translated keys for `aria-label`s on icon-only interactive utility elements.
